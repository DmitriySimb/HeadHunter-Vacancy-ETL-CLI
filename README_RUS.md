# HeadHunter Vacancy ETL CLI

Данный проект представляет собой простой ETL-пайплайн на основе shell-скриптов для сбора и обработки данных о вакансиях из API HeadHunter.

**Что такое ETL?**

ETL (Extract → Transform → Load) — это процесс работы с данными:

- Extract — получение данных (в нашем случае из API hh.ru)
- Transform — преобразование данных (JSON → CSV, очистка, сортировка)
- Load — сохранение результата (в файлы для дальнейшего анализа)

Проще говоря: мы берём “сырые” данные, приводим их в удобный вид и подготавливаем для анализа.

---

## Обзор

Проект реализует полный цикл обработки данных с использованием стандартных UNIX-инструментов.

В рамках пайплайна происходит следующее:

1. Получаем вакансии с hh.ru  
2. Сохраняем их в JSON  
3. Преобразуем в CSV (удобный формат для анализа)  
4. Сортируем данные  
5. Выделяем уровень позиции (Junior / Middle / Senior)  
6. Считаем простую статистику  
7. Разбиваем данные по датам  
8. Собираем их обратно  

---

## Быстрый старт

Запуск всего пайплайна одной командой:

```sh
./scripts/run_pipeline.sh "Data Scientist"
```

## Требования

Для работы нужны стандартные утилиты:

- POSIX shell (`sh` или `bash`)
- curl
- jq (для работы с JSON)
- awk
- sed
- sort
- uniq

Проверить:

```sh
which curl jq awk sed sort uniq
```

## Как работает пайплайн (простыми словами)

### 1. Получение данных

Скрипт делает запрос к API HeadHunter и сохраняет вакансии в JSON.  
Это “сырая” форма данных — удобна для машин, но не для анализа.

---

### 2. Преобразование в CSV

JSON превращается в таблицу (CSV), где каждая строка — это вакансия.

Мы оставляем только нужные поля:

- id
- дата создания
- название
- наличие тестового задания
- ссылка

---

### 3. Сортировка

Данные сортируются:

- сначала по дате
- затем по id

Это упрощает дальнейшую работу с ними.

---

### 4. Очистка и извлечение уровня позиции

Из названия вакансии выделяется уровень:

- Junior  
- Middle  
- Senior  
- или -, если уровень не указан  

Например:

```text
"Senior Data Scientist" → "Senior"
```

---

### 5. Подсчёт статистики

Считается, сколько вакансий каждого уровня:

```text
Junior — 8
Middle — 1
- — 11
```

Это даёт общее понимание рынка.

---

### 6. Разделение данных по датам

Данные разбиваются на отдельные файлы по дате:

```text
2025-10-22.csv
2025-10-23.csv
...
```

Это удобно, если данных становится много.

---

### 7. Обратная сборка

Все файлы объединяются обратно в один.  
Это проверка того, что разбиение прошло корректно.
